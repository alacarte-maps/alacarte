4 # What is alaCarte? #
5 
6 [![screenshot of Karlsruhe](https://github.com/alacarte-maps/alacarte/raw/master/screenshot.png "Karlsruhe")](https://alacarte-maps.github.io/stories/demo.html)
7 
8 alaCarte is a tile renderer for OpenStreetMap data written in C++11, using Cairo for
9 rendering and Boost-Spirit for [MapCSS][] parsing.
10 
11 The rendered tiles are served over HTTP using the [Slippy map tilenames][] convention.
12 
13 To compute which data is needed for rendering a tile, alaCarte uses a variant of
14 a STR-Tree.
15 
16 alaCarte was designed with medium dataset size in mind. On a typical machine with
17 at leat 8GB RAM, alaCarte can handle a unfiltered export from the federal state
18 of Baden-Wuerttemberg (Germany).
19 
20 alaCarte was developed as part of a lab course student software project at [KIT][].
21 For the old project files (mostly in German), see the [old project repository][].
22 
23 [MapCSS]: https://wiki.openstreetmap.org/wiki/MapCSS
24 [Slippy map tilenames]: https://wiki.openstreetmap.org/wiki/Slippy_map_tilenames
25 [KIT]: https://algo2.iti.kit.edu
26 [old project repository]: https://bitbucket.org/TheMarex/alacarte

35 ## Features ##
36 
37 * easy to use
38 * most MapCSS attributes are implemented (see [MapCSS wiki page][] for comparison)
39 * no need to filter OSM exports, you have full access to all attributes at runtime
40 * stylesheets are updated at runtime (changes are detected automatically)
41 * tiles can be rendered in groups ("meta tile") to speed up rendering
42 
43 [MapCSS wiki page]: https://wiki.openstreetmap.org/wiki/MapCSS/0.2#Vocabulary

46 # How to build #
47 
48 ```bash
49 git clone git@github.com:alacarte-maps/alacarte.git
50 mkdir alacarte/build
51 cd alacarte/build
52 cmake .. -DCMAKE_BUILD_TYPE=Release
53 # this starts one job per available core to build alacarte n times faster.
54 # Just run “make” without parameters if you don’t want that or run into problems.
55 make -j $(nproc)
56 ```

76 ## Dependencies ##
77 * Cairo (>=1.12.0)
78 * Boost (>= 1.55) (Spirit)
79 * libpng
80 * cmake (>= 2.8.8)
81 * asciidoc (>= 8.6.9) (optional, needed for manpage generation)
82 * DejaVu font (this is the default font if no other font is specified).
83  * If your characters are not supported by DejaVu, try to use the Noto font.
84  See [data/mapcss/default.mapcss](data/mapcss/default.mapcss) for usage examples.
85 
86 alacarte's dependency policy is to depend on nothing that's not included or newer than Debian stable.
87 So currently, we target and test on Debian Jessie, but we're open to support any other OS.

89 ## Usage ##
90 
91 See the manpages of
92 [alacarte-maps-importer](https://alacarte-maps.github.io/alacarte/manpages/alacarte-maps-importer.1.html)
93 and

95

97 to view the rendered tiles.

101 ## Install ##
102 
103 (as root)
104 
105  make install
106 
107 The config file is located in `/etc/alacarte-maps.conf`. You should make sure the user
108 running alacarte has permissions to write to the specified directories for caching
109 and logging.
